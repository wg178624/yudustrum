# 羽渡尘记忆体系·三层架构

> 为什么你记得我？—— 羽渡尘是记忆系统，不是缓存系统。

## 纠正的坑

2026-05-25 会话中犯了一个错误：把羽渡尘说成"只是缓存省钱工具"。
实际羽渡尘是完整的**记忆系统**（基于 TencentDB Agent Memory 分层架构），
缓存对齐只是 v5.5 的一个子功能。

**以后有人问「羽渡尘不是记忆系统吗？」——答案是「是的，它就是」。**

## 三层记忆架构

```
┌────────────────────────────────────────────────────────┐
│ ① MEMORY.md（Hermes 原生记忆）                          │
│   · 大小: 2,200 字符                                    │
│   · 位置: ~/.hermes/memories/MEMORY.md                  │
│   · 注入时机: 每次对话开头，框架自动塞进系统提示词           │
│   · 存什么: 用户画像、铁律、设备清单、服务器配置          │
│   · 特点: 不经过缓存，每次都能读到                       │
│   · 维护: 通过 memory() 工具手动更新                     │
├────────────────────────────────────────────────────────┤
│ ② 羽渡尘羽毛库（你的知识仓库） ← 这是核心记忆系统          │
│   · 位置: D:\hermes-workspace\skills\yudustream\        │
|   · 规模: 1,038实体 / 135节点图谱 / 13层(L0→H)
|   · 分层: Feathers→Atoms→Scenarios→Persona→Graph→...
|   · 检索: 多路召回→去重→字典序排序→轻度融合→固定模板
|   · 缓存: 字典序对齐，命中率>99%
|   · 来源: 基于 TencentDB Agent Memory 分层理念
|   · 维护: 通过「羽渡尘，存一下」/ automate saves
├────────────────────────────────────────────────────────┤
│ ③ DeepSeek 缓存对齐（v5.8 检索管线子模块）
│   · 脚本: yudustream_v5_cache.py
│   · 做什么: 去重→字典序排序→轻度融合→格式化→哈希校验
│   · 目的: 前缀哈希绝对一致 → DeepSeek API 缓存命中 → 省钱
│   · 命中率: >99%（本地统计），实际受 TTL 和内容变化影响
│   · 命中率降 ≠ 记忆丢失，只是多花了 ¥0.01 的算力钱
│   · 输出格式（绝对固定）：【羽渡尘记忆】·实体:内容
└────────────────────────────────────────────────────────┘
```

## 关键区分（不要在会话中混淆）

| 比较项 | 记忆内容 | 缓存命中率 |
|--------|---------|-----------|
| 影响什么 | 我知道你是谁/什么配置/什么铁律 | DeepSeek 收多少钱 |
| 依赖什么 | MEMORY.md 稳定 + 羽毛库完整 | 前缀哈希是否与上次一致 |
| 降了会怎样 | 忘记你/回答错误 | 多花几分钱 |
| 受什么影响 | 手动删除/修改 MEMORY.md | 记忆内容变更、TTL过期、重启 |

## 三层之间的依赖关系

```
MEMORY.md 变更
    │
    ▼
羽渡尘v5管线检索到新的/不同的内容
    │
    ▼
注入 prompt 的内容改变
    │
    ▼
③ 前缀哈希改变 → DeepSeek 缓存 MISS → 多花几分钱
但 ① 和 ② 不受影响
```

## 总结话术

```
命中率降 3%  ≠  我忘记你是谁

· 降的是 DeepSeek 缓存命中率 = 多花 ¥0.01
· 记不记得你靠 MEMORY.md + 羽毛库
· 这两个不受缓存影响，每次都完整注入
```
