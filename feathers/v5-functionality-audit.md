# 羽渡尘功能实用度审计（2026-05-23）

## 触发场景

用户在迭代全览中发现羽渡尘从 v4.3 后功能看起来像"集邮"，怀疑这些功能是否真正在生产中起作用。经讨论确认：

## 关键发现

### 1. 95% 的缓存命中率来自 DeepSeek 自己，不是羽渡尘

每轮对话 DeepSeep 自动缓存历史消息前缀，这是它自己的"对话历史缓存"机制。羽渡尘的缓存对齐（v5.5）优化的是**跨会话/跨主题**的检索前缀稳定性，而非同一会话内的历史复用。

### 2. v5.x 功能在分散话题下几乎不工作

用户日常问题（Chrome 配置、VPS 密码、N8N 研究、力扣评估）话题过于分散，导致：
- GraphRAG 图谱无关联实体可以串联
- 反馈权重每个实体命中次数 ≤3，一直处于预热状态
- 时间窗口无知识迭代更新
- 密度蒸馏始终走 DEFAULT_DENSITY (0.7)

### 3. v5.x 功能在深度领域学习中会全部激活

如果用户说"今天学 PN 结"：
- 图谱：自动拉出 扩散→耗尽层→势垒电容→击穿→整流 关联链
- 权重：PN结反复命中 → 3分 → 自动排前
- 时间窗口：记错的旧知识自动过期，新正确知识代替
- 密度：公式推导（1.0），概念概览（0.4），自动适配

### 4. 用户评价体系

用户引入了"从夯到拉"作为中文互联网 Tier 分级评价体系：
- 夯爆了 = 天花板/S级/牛逼到爆炸
- 很夯 = A级
- 还行 = B级
- 一般 = C级
- 拉了 = D级
- 拉完了 = E级/垫底/没救了

## 审计结论

不是功能没用，是使用场景没对。羽渡尘是一套为"深度学习"设计的系统，不是为"闲聊"设计的。
